EX180 - Red Hat Specialist in Containers and Kubernetes

Published on Apr 25, 2022
Updated on Apr 25, 2022

I took the EX180 certification few weeks ago.
I was really hard for me just because exam provides 2 hours that are just fine to do all things (I mean that there is no lot of time to review or retake some objectives). I hope that these tips can help someone.

Disclaimer\
This site is not affiliated with Red Hat in any way. All the information provided here are from my personal experience.

How is structured a Red Hat exam?

There are some point to note about the exam:

  • time available to work on all tasks is equal to 2 hours.
    This time also include an optionaly break (max 10’), chats or room controls with remote guy and guideline reading
  • there is no automatic system that automatically check what you do during the exam (so if you reached what the task asked for): the only way is to check yourself using command line
  • keep attention: a task can be striclty connected to a previous one so it means that a wrong result in a task can affect another one

Environment

  • In order to access exam environemnt you first need to create a usb key formatted following guidelines provided by RedHat. When your usb key is ready, plug the key and reboot in order to boot from the key itself
  • first do a compatibility check, in order to see if all things works well (connection, speakers, cams, etc.)
  • 15’ prior to exam, boot to usb key and access to the exam environment
  • you will be logged into a RHEL8 environment with Gnome DE:
    • Alt-Tab does not work. In order to move between browser (pre-installed Firefox) and terminal just click on top-left corner
    • only gedit and vim are available as text editor
    • environment will automatically open firefox on a specific page, on which there are:
      • exam guidelines
      • useful tips about linux (like moving file, change permissions, etc.)
      • objectives (on the bottom) that you need to solve in order to pass exam
    • exam will ask you to use lot of parameter: in order to avoid some mistakes (that will move you to a wrong results) copy-paste items from the browser.
      Copy from browser as usual (right-click and copy command or CTRL-C), then pasta into terminal using CTRL-SHIFT-V
    • objectives requires to create bash script in specific folder, that already contains dockerfile. Do not use them in order to follow objective
    • always follow instructions only from browser, just becuase instruction in script\dockerfile sometimes can be different and push you to a bad result

Objectives

  1. You will be asked to manage a dockerfile; be sure that build works well, because q2-q5 depends on it.
    Put missing commands\parameters in the correct position, in order to avoid a broken build.
    Test only build: do not create container or something else (it will be asked you in the next objectives)
  2. You will be asked to create\manage bash scripts that depends on q1 dockerfile.
    Objective does not require you to launch script, but you can do that just to be sure that it works\
  3. It ask to manage a db image, using bash scripts
  4. It depend on q3.
    It ask to manage an application image using bash scripts
  5. It depends on q1-q2.
    It ask to apply tag, push image to a registry and save it to a tar file
  6. It ask to deploy a db using template and specific parameters
  7. It depends on q6.
    It ask to deploy an application that works using db on q6
  8. It depends on q6-q7.
    It ask to troubleshoot application that does not work very well

Some additional tips:

  • q1 to q5 require deep knowledge on podman tool
  • q6 to q8 require dee√® knowledge on oc tool
  • if a command (from cli or bash scripts) need to manage a file, it needs file itself in the same workin folder, so be sure to move files where required
